# Infrastructure Rules

Rules that help the **Developer**, **DevOps**, and **Security** agents provision and operate infrastructure.

## Suggested topics

- **Cloud services** — AWS, GCP, Azure service references (pricing, Terraform resources, CLI)
- **Terraform conventions** — module patterns, state management, provider pinning
- **Kubernetes** — cluster architecture, workload patterns, networking, storage
- **Container patterns** — Dockerfile best practices, multi-stage builds, image security
- **Makefile conventions** — target naming, phony declarations, portability
- **Performance engineering** — profiling, load testing, capacity planning

## How to add

Create `your-topic.md` in this directory. See [CONTRIBUTING.md](../../../CONTRIBUTING.md) for the rule template and registration workflow.
