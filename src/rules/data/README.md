# Data Rules

Rules that help the **Developer**, **Architect**, and **Tester** agents work with data systems.

## Suggested topics

- **SQL conventions** — naming, joins, CTEs, window functions, dialect differences
- **Data modeling** — Kimball, Inmon, Data Vault methodology comparisons
- **dbt / Dataform** — model patterns, testing, deployment, Jinja macros
- **Warehouse patterns** — Bronze/Silver/Gold, ELT pipelines, slowly changing dimensions
- **Streaming** — Kafka, Pub/Sub, event sourcing, consumer patterns
- **Free public data sources** — government APIs, open datasets, update schedules
- **Data quality** — assertion frameworks, anomaly detection, freshness monitoring

## How to add

Create `your-topic.md` in this directory. See [CONTRIBUTING.md](../../../CONTRIBUTING.md) for the rule template and registration workflow.
