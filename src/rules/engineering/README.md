# Engineering Rules

Rules that help the **Developer**, **Architect**, and **Tester** agents write better code.

## Suggested topics

- **Language conventions** — Go, Python, TypeScript, Rust style guides and idioms
- **API design** — REST, gRPC, GraphQL contract patterns
- **Protobuf / schema design** — message layout, versioning, backward compatibility
- **Testing strategy** — unit vs integration vs E2E, coverage thresholds, test doubles
- **CI/CD patterns** — pipeline stages, artifact management, release workflows
- **Observability** — structured logging, metrics, tracing conventions (OpenTelemetry)

## How to add

Create `your-topic.md` in this directory. See [CONTRIBUTING.md](../../../CONTRIBUTING.md) for the rule template and registration workflow.
