# Regulatory Rules

Rules that help all agents build software that is not just great, but also **compliant**.

## Suggested topics

- **GDPR** — data processing, consent, data subject rights, cross-border transfers
- **DORA** — ICT risk management, incident reporting, third-party oversight (EU financial sector)
- **AI Act** — risk classification, transparency requirements, prohibited practices
- **SOX IT controls** — change management, access controls, audit trails
- **PCI-DSS** — payment card data handling, network segmentation, encryption
- **Software licensing** — OSI-approved licenses, SPDX identifiers, copyleft vs permissive
- **Accessibility** — WCAG 2.1, ADA Section 508, inclusive design patterns

See [CONTRIBUTING.md](../../../CONTRIBUTING.md#adding-regulatory-and-legal-rules) for acceptance criteria — every regulatory rule must be publicly sourced, commercially permissive, broadly applicable, and actionable for engineers.

## How to add

Create `your-topic.md` in this directory. Follow the regulatory rule template in CONTRIBUTING.md.
